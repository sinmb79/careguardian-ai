# CareGuardian AI 비공개 테스트 운영 가이드

- 대상 버전: Android `1.0.1` (`versionCode 6` 후보)
- 테스트 트랙: Google Play 비공개 테스트 `Alpha`
- 데이터 범위: **합성 데이터만 허용**
- 운영 목표: 최소 12명이 참여 동의 상태를 14일 연속 유지하면서 기능·안전·접근성 피드백 수집

> 이 앱은 의료기기, 복약 확인 수단 또는 응급 연락 수단이 아닙니다. 실제 사람, 실제 약, 처방, 건강정보, 전화번호와 긴급연락처를 입력하지 마세요.

## 1. 참여 전에 확인할 것

1. 테스터는 Play 초대에 사용한 Google 계정으로 로그인합니다.
2. 테스트 링크에서 참여를 선택한 뒤 Google Play에서 설치합니다. APK나 AAB를 메신저로 배포하지 않습니다.
3. 화면 잠금과 기기 인증(지문·얼굴·기기 자격증명)을 설정합니다.
4. 아래 합성 데이터 예시만 사용합니다.

| 항목 | 허용 예시 | 금지 예시 |
|---|---|---|
| 이름 | `테스트 사용자 A` | 본인·가족 실명 |
| 약 | `가상약 A` | 실제 처방약·영양제 이름 |
| 연락처 | `000-0000-0000` | 실제 전화번호 |
| 안정화·루틴 | `가상 아침 일정` | 실제 장애·질환·행동 정보 |

## 2. 핵심 테스트 시나리오

각 시나리오는 성공·실패 여부와 화면에 보인 정확한 안내 문구를 기록합니다.

| ID | 시나리오 | 기대 결과 |
|---|---|---|
| CT-01 | 첫 실행 | 빈 프로필 화면이 열리고 실제 데이터 금지 안내와 개인정보처리방침 링크를 확인할 수 있음 |
| CT-02 | 약 이름만 입력 | 용량·시각·복용법이 자동으로 만들어지지 않음 |
| CT-03 | 잘못된 시각 입력(`25:99`, `8:3`) | 알림이 예약되지 않고 성공으로 표시되지 않음 |
| CT-04 | 복약 정보 저장 시도 | 별도 동의를 하기 전에는 저장이 중단됨 |
| CT-05 | 알림 권한 거부 후 저장 | 프로필 저장 결과와 알림 실패가 구분되어 안내됨 |
| CT-06 | 알림 권한 허용 후 `HH:mm` 저장 | 민감정보 없는 로컬 알림이 매일 예약되며 예약 건수를 안내함 |
| CT-07 | 백그라운드 전환·재진입 | 돌봄 화면이 즉시 잠기고 인증 전에는 이름·약·연락처가 보이지 않음 |
| CT-08 | 보호자 편집 열기 | 저장된 프로필에서는 기기 인증을 요구함 |
| CT-09 | 스크린샷·최근 앱 화면 | 민감 화면 캡처가 차단됨. 접근성 도구 충돌이 있으면 기록 |
| CT-10 | 앱 강제 종료·재시작 | 기기 인증 성공 후에만 암호화된 프로필이 열림 |
| CT-11 | 복약 정보 변경 | 변경 후 별도 동의를 다시 요구함 |
| CT-12 | 전체 삭제 | 예약 알림 취소 확인 후 프로필·암호화 DB·키가 삭제되고 새 프로필 화면으로 돌아감 |
| CT-13 | 삭제 후 재시작 | 삭제한 프로필이 복구되지 않음 |
| CT-14 | 알림 지연·누락 관찰 | 제조사·절전·DND 상태와 함께 기록하며 실제 복약 판단에는 사용하지 않음 |

## 3. 14일 운영표

| 기간 | 확인 내용 |
|---|---|
| 참여일(D0) | 12명 이상 opt-in 확인, 설치·첫 실행·정책 링크 확인 |
| D1–D3 | 저장·잠금·인증·알림 권한 허용/거부·잘못된 시각 테스트 |
| D4–D7 | 재부팅, 절전 모드, DND, 백그라운드와 알림 지연 관찰 |
| D8–D11 | 편집 재인증, 동의 재요구, 전체 삭제와 재설치 검증 |
| D12–D14 | 회귀 재확인, 접근성·문구 피드백, 중대한 문제 미해결 여부 판단 |

참여자 수는 Play Console의 참여 상태를 기준으로 매일 확인합니다. 링크를 열어 보기만 한 사람이나 테스트 참여를 철회한 사람은 유지 인원으로 보지 않습니다.

## 4. 피드백 양식

실제 개인정보나 화면 캡처를 첨부하지 않습니다.

```text
테스트 ID: CT-__
앱 버전: 1.0.1 (6)
기기 제조사/모델:
Android 버전:
발생 시각과 시간대:
알림 권한 / 절전 / DND 상태:
재현 단계:
기대 결과:
실제 결과:
재현 빈도:
접근성 도구 사용 여부:
```

## 5. 즉시 중단 기준

다음 중 하나라도 발생하면 새 설치와 테스트를 중단하고 해당 릴리스를 비활성화합니다.

- 인증 전에 저장된 이름·약·연락처가 표시됨
- DB 또는 로그에서 합성 입력값이 평문으로 노출됨
- 알림 잠금화면·payload에 약 이름이나 연락처가 표시됨
- 삭제 성공 안내 뒤 프로필이나 CareGuardian 예약 알림이 남음
- 앱이 입력하지 않은 용량·복용법·돌봄 지시를 만들어 냄
- 저장·알림 실패를 성공으로 안내함
- 광고·분석·원격 알림 초기화 등 선언하지 않은 네트워크 전송이 확인됨
- 크래시, 데이터 손상 또는 인증 불능이 반복됨

## 7. 아직 허용하지 않는 범위

- 실제 개인정보·민감정보·복약정보
- 실제 복약 이행 판단이나 누락 감지
- 응급 호출, 보호자 자동 연락, 진단·처방·상호작용 판단
- 루팅 기기, 업무용 MDM 기기, 화면 잠금이 없는 기기
- iOS/TestFlight 테스트

실제 데이터 단계는 실제 기기 포렌식, 프록시·DNS 네트워크 관찰, Samsung/Pixel 알림 매트릭스, 장애 대응과 삭제 증거, 당사자·돌봄 파트너 공동설계를 별도 통과한 뒤 다시 심사합니다.
